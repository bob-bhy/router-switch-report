快速结论

MicroTech 是一家成立于 2017 年、在巴库从事设备销售、系统集成及软件/云与托管服务的本地 ICT 公司。它既运营自有平台（如 mcloud），也作为厂商经销商为各行业企业提供解决方案，近期在本地商会活跃并积极宣传 Fortinet 等安全产品。

客户的目标市场（行业 / 地域 / 规模类型）

  • 行业：金融、医疗、制造/生产、运输/车队、餐饮/零售、政府/公用事业等。 [Microtech] [scan.az]
  • 地域：以阿塞拜疆本土（巴库 / Nizami 区）为主。 [Microtech]
  • 规模类型：目标客户以中小型到中大型企业为主。 推断：从其提供“ERP、Cloud、Infrastructure、Integration”产品线可推断 [Microtech]

客户的业务模式

  • 核心业务：设备销售、系统集成、IT 项目/咨询、安全系统、组装/安装、培训与网络服务。 [Microtech]
  • 自研/自营产品：运营自有云/托管服务 (mcloud.az) 及多个子项目/平台 (cartrack.az, scan.az)。 [mcloud.az] [scan.az]
  • 渠道：既作为厂商经销商/代理，也直接面向终端提供集成和托管服务。 [Kenwood] [Zeta Alarms]

市场地位

阿塞拜疆本地活跃的 SI / IT 解决方案提供商，自述“113 客户 / 287 个项目”。 [Microtech]

公司成立时间

合作可能性与风险分析

潜在机会

  • 需求高度契合：其业务直接需要交换机、防火墙、无线 AP 等网络与安全设备。推断
  • 有代理经验：愿意并有能力代理/交付厂商级产品，并已在宣传 Fortinet。推断

潜在风险

  • 厂商授权不透明：需验证其官方合作伙伴级别。
  • 财务能力未知：大额项目需信用审查。推断
  • 市场竞争激烈：本地市场可能价格敏感。推断
  • 跨境操作复杂：物流、关税、保修需明确。推断
  • 区域通用风险：政治与汇率波动。推断

潜在痛点（推测）

  • 需要可靠的本地/厂商技术支持与售后。
  • 对“授权/证书/技术能力”的证明需求。
  • 价格与交货周期的压力。
  • 对安全产品持续更新、license 管理与培训支持的需求。

推荐的销售 / 跟进动作 (给 Bob)

1. 第一封跟进邮件核心要点：

  • 精准切入：提及他们的官网项目 (mcloud, cartrack) 或 Fortinet 宣传，展示你已做功课。
  • 价值主张：强调你能提供稳定货源、本地支持或 PoC 设备。
  • 明确请求：直接询问采购联系人信息、流程，以及他们当前的厂商代理级别。

2. 优先电话/WhatsApp 确认三件事：

  • 是否为厂商正式授权经销商（厂商、级别）。
  • 近期是否有大型项目在推进（时间、预算）。
  • 是否需要本地技术培训或 PoC 设备。

3. 商业提案建议：

  • 提供「设备 + 本地安装 + 支持」的整包报价，突出 SLA。
  • 针对安全产品，明确 License 周期与续费策略，并提供培训选项。

4. 信任建设与风险控制：

  • 提供区域案例、授权证书或参考客户来建立信任。
  • 对大额订单要求 PO 与预付款，或使用信用证。
